# f00 MVP Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship a cross-platform `f00` binary that lists directories with solid core behavior, common GNU flags, TTY colors/icons, optional git status, TOML config, CI, install script, and an accurate f00.sh landing page — without implementing post-MVP TUI/archives/plugins.

**Architecture:** Approach B — layered crates (`f00-core` → `f00-format` / `f00-compat` / `f00-git` → thin `f00-cli`). Default Cargo features enable git + icons + machine formats. GNU dialect everywhere; modern TTY defaults; strict mode via `--gnu` / `F00_GNU`.

**Tech Stack:** Rust (edition 2021+), Cargo workspace, clap 4, serde/toml, thiserror, git2 (feature), owo-colors (or equivalent), insta for snapshots, GitHub Actions, static GitHub Pages site, POSIX `scripts/install.sh`.

**Design reference:** [docs/superpowers/specs/2026-07-16-f00-design.md](../specs/2026-07-16-f00-design.md)

**Binary name:** `f00`  
**Crate names:** `f00-core`, `f00-format`, `f00-compat`, `f00-git`, `f00-cli`
