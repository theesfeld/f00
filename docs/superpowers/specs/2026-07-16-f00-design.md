# f00 Design Specification

**Date:** 2026-07-16  
**Status:** Approved (product decisions locked)  
**Domain:** [f00.sh](https://f00.sh)  
**Binary name:** `f00` (also installable as `ls`)

---

## 1. Product Vision

**f00** is a modern, cross-platform directory listing tool and a drop-in replacement for GNU `ls`, written in Rust. It ships three faces of the same product (**ACD**):

| Face | Description |
|------|-------------|
| **A — Drop-in `ls`** | GNU-style flags, path arguments, and exit semantics so `f00` can replace `ls` via symlink or `alias ls=f00` without breaking scripts. |
| **C — Modern `f00` CLI** | TTY-aware defaults (colors, icons, human sizes, git status), TOML config, themes, and ergonomic long options. |
| **D — Supertool** | Kitchen-sink ambition: tree view, JSON/CSV/TSV, archives listing, TUI browser, plugins — quality bar for v1 is **core + git + icons**. |

**Tagline:** *list everything, break nothing.*

### 1.1 Non-goals (MVP)

- Full plugin host ABI (post-MVP)
- Interactive TUI file manager as default path (optional crate, later)
- Network filesystem remote protocols as first-class (treat as normal paths via OS)
- Replacing `find`/`fd` or recursive search tools
- Perfect byte-identical GNU `ls` output in every locale without `--gnu` (strict mode is opt-in)

### 1.2 Success criteria

1. On Linux/macOS/FreeBSD/Windows, `f00` lists a directory correctly with stable sorting and clear errors.
2. Common GNU flags (`-l`, `-a`, `-A`, `-h`, `-R`, `-1`, `-C`, `-t`, `-S`, `-r`, `-d`, `-F`, color-related) work.
3. In a TTY with default features: colors + icons + git status column when inside a git work tree.
4. `cargo test` and CI green on all four platform families.
5. `curl -fsSL https://f00.sh/install.sh | sh` installs a working binary.
6. README and GitHub Pages site stay accurate and “chromed” (polished marketing + real command output).

## 2. Architecture (Approach B)

**Layered library crates + thin CLI.** Business logic lives in libraries; `f00-cli` is wiring (clap, config load, feature gates, exit codes).

```
┌─────────────────────────────────────────────────────────────┐
│                         f00-cli                              │
│  clap · TOML config · env · install-as-ls · exit codes      │
└────────────┬──────────────────┬──────────────────┬──────────┘
             │                  │                  │
             ▼                  ▼                  ▼
      ┌────────────┐    ┌─────────────┐    ┌────────────┐
      │ f00-compat │    │  f00-format │    │  f00-git   │
      │ GNU flags  │    │ columns ·   │    │ status map │
      │ --gnu mode │    │ colors ·    │    │ (feature)  │
      │            │    │ icons ·     │    │            │
      └─────┬──────┘    │ themes ·    │    └─────┬──────┘
            │           │ tree/json   │          │
            │           └──────┬──────┘          │
            │                  │                 │
            └────────────┬─────┴─────────────────┘
                         ▼
                  ┌────────────┐
                  │  f00-core  │
                  │ Entry ·    │
                  │ readdir ·  │
                  │ sort/filter│
                  │ platforms  │
                  └────────────┘
```

Optional later crates (not in default binary):

- `f00-tui` — interactive browser
- `f00-archive` — list inside zip/tar/etc. without extract

### 2.1 Crate responsibilities

#### `f00-core`

- Platform-agnostic **Entry** model (name, path, file type, size, mtime, mode/permissions, uid/gid or owner strings, symlink target, device ids where available).
- **Directory reading** via a small platform trait (`ReadDirExt` / `MetadataExt`) with OS-specific impls (Linux, macOS, FreeBSD, Windows).
- **Filter** (dotfiles, only dirs, ignore patterns later).
- **Sort** (name, size, mtime, extension; reverse; directories-first option).
- Recursive walk for `-R` with cycle/error handling.
- No formatting, no clap, no git.

#### `f00-format`

- Layout engines: single column (`-1`), multi-column (`-C`), long (`-l`), tree, across.
- Colorization (ANSI) driven by theme + file type + extension map (LS_COLORS-compatible subset + f00 theme).
- Icon mapping (Nerd Font glyphs by default; `--no-icons` / config off).
- Human-readable sizes (`-h`).
- Machine formats: JSON, CSV, TSV (stable schemas versioned).
- Does not read the filesystem except for what Entries already carry.

#### `f00-compat`

- GNU `ls` flag surface mapping into a shared `ListRequest` / options struct.
- **Strict `--gnu` mode** and env `F00_GNU=1`: force GNU-like defaults (no icons, classic color rules, classic time format, classic quoting).
- Documented intentional divergences outside strict mode.
- Compatibility tests against golden fixtures where feasible.

#### `f00-git` (default feature)

- Discover git work tree root for listed paths.
- Status map (index + worktree) keyed by path: modified, staged, untracked, ignored, conflicted, typechange, renamed.
- Efficient enough for large repos: prefer libgit2 (`git2`) or a thin porcelain fallback; cache per process for one invocation.
- Graceful degrade if not a repo or git unavailable (feature compiled but no .git).

#### `f00-cli`

- Binary name **`f00`**.
- Clap derive CLI; argv0 detection for `ls` symlink behavior (slightly more GNU-default when invoked as `ls`).
- Load TOML config from XDG / platform dirs; merge: defaults < config < env < flags.
- Feature-gated modules (git, icons already in format, tui/archive later).
- Exit codes: `0` success, `1` minor problems (e.g. unreadable entry), `2` serious (usage / fatal) — GNU-aligned.

### 2.2 Data flow

1. Parse CLI + config → `AppOptions`.
2. Resolve paths (default `.`).
3. For each path: `f00-core` produces `Vec<Entry>` (or stream for huge dirs if needed later; MVP can collect).
4. Optional: `f00-git` annotates entries with `GitStatus`.
5. `f00-format` renders to stdout/stderr (errors on stderr).
6. Aggregate exit code.

### 2.3 Feature flags (Cargo)

| Feature | Default | Purpose |
|---------|---------|---------|
| `git` | yes | `f00-git` + git column |
| `icons` | yes | icon tables in `f00-format` |
| `formats` | yes | json/csv/tsv |
| `tui` | no | `f00-tui` |
| `archives` | no | `f00-archive` |
| `plugins` | no | dynamic extensions (post-MVP) |

**Default binary** = core + git + icons + formats. Distributors can build `f00-cli` with `--no-default-features` for minimal.

## 3. Cross-Platform Strategy

| Platform | Notes |
|----------|--------|
| **Linux** | Primary; full mode bits, xattrs later, LS_COLORS. |
| **macOS** | BSD-ish metadata; same GNU flag dialect (product decision: GNU-style flags everywhere). |
| **FreeBSD** | Unix metadata path shared with Linux/macOS via `cfg` modules. |
| **Windows** | Use `std::fs` + Windows metadata; map permissions display to a readable long format; support `%LOCALAPPDATA%\f00\config.toml`; path separators normalized in display optionally. |

Platform-specific code isolated under `f00-core/src/os/{unix,windows}.rs` (and finer splits as needed). Public API stays portable.

**CI matrix:** Ubuntu, macOS, Windows (GitHub Actions); FreeBSD via cross or community runner when available, plus documented manual smoke.

## 4. CLI Dialect

### 4.1 Defaults

| Context | Behavior |
|---------|----------|
| TTY stdout | Colors on; icons on (if feature); multi-column; git column when repo + feature |
| Pipe/file | No color unless forced; no icons; single-column or appropriate non-TTY layout |
| Invoked as `ls` | Prefer GNU-compatible defaults more aggressively; still allow f00 enhancements unless `--gnu` |
| `--gnu` or `F00_GNU=1` | Strict GNU-like: disable icons/git decorations that break scripts; classic fields |

### 4.2 Flag surface (MVP priority)

**Must-have (core GNU):**

- `-a` / `--all`, `-A` / `--almost-all`
- `-l` (long), `-1`, `-C`, `-x` (across) as layout allows
- `-h` / `--human-readable`
- `-r` / `--reverse`, `-t` (mtime), `-S` (size), `-U` (dir order) if cheap
- `-R` / `--recursive`
- `-d` / `--directory`
- `-F` / `--classify`
- `-G` (macOS users expect no-group; map carefully — on GNU, `-G` is no-group; document)
- `--color[=WHEN]`, `--icon[=WHEN]`
- `-n` numeric uid/gid where applicable

**Modern f00 (always available):**

- `--tree` / `-T`
- `--git` / `--no-git`
- `--icons` / `--no-icons`
- `--theme NAME`
- `--json` / `--csv` / `--tsv`
- `--config PATH`
- `--gnu`
- `--hyperlink` (optional OSC 8 if low risk)

### 4.3 Config (TOML only)

Search order (first found wins as user config; system optional later):

1. `$F00_CONFIG` if set
2. Linux/BSD: `$XDG_CONFIG_HOME/f00/config.toml` or `~/.config/f00/config.toml`
3. macOS: `~/Library/Application Support/f00/config.toml` (also accept XDG if set)
4. Windows: `%LOCALAPPDATA%\f00\config.toml`

Example schema:

```toml
[defaults]
all = false
long = false
human = true
icons = "auto"      # auto | always | never
color = "auto"
git = "auto"
theme = "default"
classify = false

[sort]
by = "name"         # name | size | time | extension
reverse = false
dirs_first = true

[format]
time_style = "iso"  # iso | long-iso | full-iso | locale | +FORMAT
quoting = "shell-escape"

[git]
enabled = true

[icons]
enabled = true
```

No YAML/JSON config formats. Env overrides: `NO_COLOR`, `CLICOLOR_FORCE`, `F00_GNU`, `LS_COLORS` (subset), `F00_THEME`.

## 5. Entry Model (conceptual)

```rust
// Illustrative — final fields live in f00-core
pub struct Entry {
    pub path: PathBuf,
    pub name: OsString,
    pub file_type: EntryType, // File, Dir, Symlink, Fifo, Socket, Block, Char, Door, Unknown
    pub meta: Option<Meta>,   // None if lstat failed
    pub symlink_target: Option<PathBuf>,
    pub git: Option<GitStatus>, // filled by f00-git
}

pub struct Meta {
    pub len: u64,
    pub modified: Option<SystemTime>,
    pub accessed: Option<SystemTime>,
    pub created: Option<SystemTime>,
    pub permissions: PermissionsDisplay, // unix mode or windows attrs
    pub nlink: u64,
    pub uid: Option<u32>,
    pub gid: Option<u32>,
    // ...
}
```

Sorting and filtering operate on `Entry`. Formatters never re-stat unless a future “follow for display” option requires it (document carefully for symlink vs lstat).

## 6. Packaging & Distribution

### 6.1 Hybrid packaging

- **crates.io / source:** workspace of `f00-*` crates; binary from `f00-cli`.
- **Default release binary:** features `git`, `icons`, `formats`.
- **Minimal builds:** document `cargo build -p f00-cli --no-default-features`.
- **Install script:** `https://f00.sh/install.sh` (mirrored in repo as `scripts/install.sh`) — bun-style:
  - Detect OS/arch
  - Download latest GitHub Release asset
  - Install to `~/.f00/bin` or `/usr/local/bin` with user choice
  - Optional `ls` symlink
  - Print PATH instructions
- **GitHub Releases:** multi-target builds from CI (linux gnu/musl, macOS universal or dual, windows msvc, freebsd if feasible).

### 6.2 Landing site (GitHub Pages)

- Repo `site/` content deployed to GitHub Pages for **f00.sh**.
- Always in sync with project: version badge, install one-liner, feature list, screenshots/asciinema, link to docs.
- Process: release workflow updates version string; PR checklist requires site + README accuracy.

### 6.3 README policy

- Chromed: logo, badges, demo GIF/SVG, quick start, flag table, config sample.
- **100% accurate:** no vapor features listed as done; mark “planned” clearly.
- Docs sync process: see implementation plan and ROADMAP.

## 7. Quality Bar & Testing

| Layer | Strategy |
|-------|----------|
| Unit | Sort/filter pure functions; theme/icon maps; flag parse matrix |
| Integration | Temp dirs with known tree; snapshot stdout (insta or similar) |
| Compat | Golden cases for `--gnu` vs fixtures |
| Git | Fixture repo in `tests/fixtures/git-repo` (or temp init) |
| Cross-platform | CI matrix; `cfg`-gated tests |
| Manual | FreeBSD smoke; Windows PowerShell install script |

**Quality bar for “done” MVP:** reliable core listing + git status + icons + common GNU flags + install path + site + CI. Not full kitchen-sink.

---

## 8. Security & Safety

- No `curl | sh` install script tricks: HTTPS only, checksum verification against release SHASUMS when published.
- Do not follow symlinks into unexpected places for `-R` without clear rules (document: physical vs logical; MVP = don’t loop, surface errors).
- Archive/TUI later: sandbox large untrusted inputs (zip bombs) when those crates land.
- Dependencies: minimize `unsafe`; audit `git2`/zlib stack.

---

## 9. Versioning & Stability

- SemVer for crates and binary.
- `0.x` until CLI and JSON schema stabilize.
- JSON schema version field: `"schema_version": 1`.
- Breaking CLI changes after 1.0 require major bump; deprecations warned on stderr when TTY.
