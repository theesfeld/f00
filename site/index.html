Pretty where it matters

On a TTY: icons, git status, modern long-format colors. Feels like a product — not a 1990s dump of names.

02

Safe in pipes

Non-TTY stdout auto-enables script-safe / GNU-equivalent mode. No icon garbage in your CI logs. Force modern with --no-gnu when you want it.

03

JSON is core

--json / -j with rich metadata. Pretty and syntax-colored when color is on; compact plain for jq when color is off.

04

Speed is the product

Parallel metadata, cheap short paths, Linux statx + io_uring. Profile with --profile.

What ships

No vapor. Optional pieces stay optional.

GNU surface + auto safety

Full flag set. Non-TTY ≈ --gnu. Escape hatch: --no-gnu.

JSON · tree · CSV

Icons · git · LS_COLORS

Nerd Font icons, git column, modern long colors — TTY smart defaults.

f00-tui

Separate dual-pane browser binary in the same release tarball.

Self-update

f00 --update / --check-update via GitHub Releases + checksums.

Lean defaults

Default cargo features: git + io-uring. Archives / plugins / embedded TUI opt-in.

Speed

Two tracks. Don’t mix them.

Track A · drop-in

coreutils ls vs f00 --gnu

Same job: no icons, no git, script-shaped output.

Track B · modern tools

eza / lsd vs product f00

Daily-driver defaults (no --gnu).

f00 --gnu · 20k files · hyperfine 20× · colors off

-l long listing

f00 wins

f00 --gnu -l38.0 ms
coreutils ls -l50.1 ms

1.32× faster than GNU ls -l.

-1 names only

coreutils still leads

coreutils ls -110.4 ms
f00 --gnu -120.1 ms

Getdents-tuned C still wins pure name dump. We’re closing the gap without stripping features.

product f00 (not --gnu) · 20k files · hyperfine 20× · colors off

-1 short

beats eza & lsd

f00 -119.0 ms
eza -131.2 ms
lsd -165.4 ms

1.6× eza · 3.4× lsd

-l long

beats eza & lsd

f00 -l39.0 ms
eza -l53.5 ms

1.4× eza · 2.6× lsd

chrome still fast

Against modern Rust tools, product f00 wins the common cases — short, long, and icon long — without turning features off.

Install

One command. Options if you already live in another ecosystem.

$ curl -fsSL https://f00.sh/install.sh | bash

Installs to ~/.local/bin · ships f00 + f00-tui · never touches /bin/ls 

# pin a release
curl -fsSL https://f00.sh/install.sh | F00_VERSION=__VERSION__ bash

# custom directory
curl -fsSL https://f00.sh/install.sh | INSTALL_DIR=$HOME/bin bash

# optional: also put ls → f00 on PATH (not /bin/ls)
curl -fsSL https://f00.sh/install.sh | F00_INSTALL_LS=1 bash
# later
f00 --update
f00 --check-update
$ nix profile install github:theesfeld/f00

First-class alongside install.sh · builds from source

nix run github:theesfeld/f00 -- -la
# or flake checkout: nix build .#default
$ cargo install f00 --locked

crates.io · lean defaults (git + io-uring). TUI: cargo install f00-tui --locked when published as bin package, or build from the repo.

git clone https://github.com/theesfeld/f00.git
cd f00 && cargo build --release -p f00 -p f00-tui

Convenience channels — may lag the GitHub release by a bit.

# Homebrew
brew install theesfeld/tap/f00

# Arch (AUR)
yay -S f00

# Scoop / winget / deb / rpm
# → https://github.com/theesfeld/f00/releases

Prefer the package manager’s own upgrade path for these installs.
