Self-update

Install in one line

Detects OS/arch, downloads the GitHub release, verifies SHA-256 when present, and installs to ~/.local/bin (override with INSTALL_DIR).

Quick install

Latest release v0.7.1 

curl -fsSL https://f00.sh/install.sh | bash

Add to your PATH if needed:

export PATH="$HOME/.local/bin:$PATH"

Pin a version or choose a directory:

curl -fsSL https://f00.sh/install.sh | F00_VERSION=v0.7.1 bash
curl -fsSL https://f00.sh/install.sh | INSTALL_DIR=$HOME/bin bash

We never replace system /bin/ls by default. The primary command is always f00.

Update

Once installed, upgrade from GitHub Releases (checksum verified):

f00 --update
# or
f00 update

Check without changing the binary (exit 1 if behind):

f00 --check-update
# or
f00 check-update

git clone https://github.com/theesfeld/f00.git

Using f00 as ls

The product command is f00. Drop-in for ls is optional — we never replace system /bin/ls by default.

1. Shell alias (recommended)

Interactive shells only. Scripts that call /bin/ls are unchanged.

# modern defaults (icons, git, …)
echo "alias ls='f00'" >> ~/.bashrc    # or ~/.zshrc
echo "alias ll='f00 -la'" >> ~/.bashrc

# coreutils-shaped (no icons/git)
# echo "alias ls='f00 --gnu'" >> ~/.bashrc
# or: export F00_GNU=1

2. PATH symlink (installer opt-in)

Puts ls → f00 next to the binary on your PATH. Does not overwrite /bin/ls. 

curl -fsSL https://f00.sh/install.sh | F00_INSTALL_LS=1 bash

3. Binary named ls

If argv0 is ls, f00 uses quieter defaults (icons / dirs-first off). Full strict mode still needs --gnu or F00_GNU=1.
